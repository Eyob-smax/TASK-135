Implement a District Resource & Inventory Operations Console for K–12 organizations to manage reading resources, physical inventory, and administrative operations in a fully offline desktop environment. The app targets Windows 11 with an English UI and a baseline of 1920x1080 resolution with high-DPI scaling, packaged as a signed .msi installer. Primary roles are Administrator (system setup and auditing), Librarian/Inventory Clerk (cataloging, counts, and adjustments), Teacher (requests and classroom allocations), Counselor (audience and timeliness classification for student-facing materials), and Reviewer/Approver (publishing and policy checks). Users sign in with locally validated username/password; passwords must be at least 12 characters and stored as salted hashes, with an automatic lockout for 15 minutes after 5 failed attempts. Desktop interaction must be keyboard-first with global shortcuts (e.g., Ctrl+F search, Ctrl+N new record, Ctrl+Shift+L open inventory ledger), right-click context menus for common actions (freeze stock, relocate, publish/unpublish), multi-window workflows (ledger + item detail + count session side-by-side), and a system tray mode that keeps scheduled tasks running while minimizing to tray. The reading resource library supports importing books, picture books, articles, and audio by selecting local files or bulk CSV; each item maintains categories, tags, difficulty level, theme, and age range, with deduplication using a file fingerprint plus ISBN/metadata matching and version control that retains the last 10 revisions. Publishing is controlled by a review workflow: Draft → In Review → Published/Unpublished, with required reviewer notes and an immutable timestamped audit trail.

Built with PyQt for a responsive desktop UI and SQLite for local persistence, the system maintains an inventory ledger at warehouse/location/batch/serial/status granularity, including freeze/unfreeze and record-level locking to prevent concurrent edits. Stock counts support open, blind, and cycle modes; a count session expires after 8 hours of inactivity and produces gain/loss adjustments requiring a reason code and supervisor approval when the variance exceeds ±$250.00 or 2% of on-hand quantity. Intra-warehouse relocation must capture from/to bin, quantity, operator, and device (manual or USB barcode scanner input), and every adjustment is append-only with reversible correction entries (no hard deletes). A centralized configuration center manages dictionary items, workflow nodes, notification templates (local only), and regional business-district descriptors used for reporting; metadata taxonomy supports multi-level categories, keywords, timeliness, source, and copyright fields with validation rules (e.g., age range must be 0–18, timeliness must be one of Evergreen/Current/Archived). RBAC enforces menu and API-level access plus data scope by school/department/class/individual. “Open platform” integrations are implemented strictly as local REST endpoints and webhook-style outbound events written to a LAN-shared folder (no internet), using request signing (HMAC-SHA256), rotating keys every 90 days, and rate limiting to 60 requests/min per client. The app must start in under 5 seconds on a standard office PC, run for 30 days without memory growth beyond 200 MB steady-state, and include crash-safe checkpointing so in-progress imports, count sessions, and approval queues resume automatically after restart; updates are applied via offline package import with version rollback to the prior build retained locally.
